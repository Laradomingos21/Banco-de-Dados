Banco de dados

O que é e para que serve

SGBD (Sistema Gerenciador de Banco de Dados) é um software responsável por criar, gerenciar e manter bancos de dados. Em inglês, DBMS (DataBase Management System). Serve para armazenar, organizar, proteger e manipular dados de forma estruturada, garantindo acesso rápido, seguro e confiável às informações dentro de uma empresa ou sistema.

Um SGBD normalmente possui:

  • Motor de banco de dados (executa as operações);
  • Subsistema de definição de dados (DDL) – define estrutura do banco;
  • Subsistema de manipulação de dados (DML) – insere, altera, exclui e consulta dados;
  • Subsistema de administração – gerencia segurança, integridade e backups;
  • Subsistema de geração de aplicações – auxilia no desenvolvimento de sistemas que usam o banco.

Funções principais

    Métodos de acesso
    • DDL – define estrutura e tipos de dados.
    • DML – manipula os dados (inserir, alterar, excluir, consultar).
    Integridade semântica
    • Garante que os dados sejam válidos e coerentes com o domínio da aplicação.
    Segurança
    • Controle de acesso por usuários e permissões.
    • Proteção contra falhas e perda de dados (logs e backups).
    Concorrência
    • Evita conflitos em acessos simultâneos, usando bloqueios (locks) ou timestamp.
    Independência
    • Física: mudanças na forma de armazenamento não afetam o uso.
    • Lógica: alterações na estrutura lógica não afetam as aplicações.

Exemplo prático

Tipos de bancos de dados

Banco de Dados Relacional (SQL)

O banco de dados relacional organiza as informações em tabelas compostas por linhas e colunas, onde cada linha representa um registro (com uma chave exclusiva) e cada coluna corresponde a um atributo. Esse modelo é baseado na lógica relacional, o que torna os dados estruturados, consistentes e de fácil acesso.

Uma de suas maiores vantagens é o uso da linguagem SQL (Structured Query Language), que permite criar, consultar e manipular dados de forma padronizada e eficiente.

Os bancos relacionais priorizam consistência, integridade e controle transacional, sendo ideais para aplicações que exigem dados estruturados e precisos, como sistemas financeiros, cadastros e gestão empresarial.

Banco de dados não relacional

O banco de dados não relacional, também chamado de NoSQL (Not Only SQL), não utiliza o modelo tradicional de tabelas. Ele permite armazenar dados de forma flexível, em modelos como chave-valor, documentos (JSON), colunas ou grafos, adaptando-se melhor a grandes volumes de dados e estruturas não padronizadas.

Surgiu como alternativa moderna para lidar com dados não estruturados ou semiestruturados, comuns em aplicações de Big Data, redes sociais e Internet das Coisas (IoT).

Sua principal força está na escalabilidade horizontal (distribuição em vários servidores) e na velocidade de leitura e escrita, o que o torna ideal para sistemas que processam grandes quantidades de informações em tempo real.

comparação entre os dois tipos

critério SQL NoSQL
estrutura tabelas com linhas e colunas e esquema rígido estruturas flexíveis como documentos, chave-valor, colunas ou grafos

escalabilidade baixa (depende de um esquema fixo) alta (permite mudanças e dados variados)

portanto os bancos relacionais priorizam consistência e estrutura, enquanto os não relacionais priorizam velocidade e flexibilidade. A escolha entre eles depende do tipo de dado e das necessidades do sistema.

Características e Importância no Desenvolvimento de Sistemas

A escolha entre banco de dados relacional e não relacional impacta diretamente o desempenho, a escalabilidade e a confiabilidade de um sistema.

  • Relacionais (SQL):
    • Garantem integridade e consistência rigorosa dos dados (transações ACID).
    • São ideais para sistemas corporativos, com dados interligados e estruturados.
    • Facilitam o controle de relacionamentos complexos entre entidades.
    • Usam SQL para consultas padronizadas e relatórios precisos.
  • Não Relacionais (NoSQL):
    • Oferecem maior desempenho e flexibilidade em ambientes dinâmicos.
